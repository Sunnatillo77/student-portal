# Технический стек проекта "Студенческий портал"

## Frontend

### Основные технологии

- **React 19.2.0** - JavaScript библиотека для создания пользовательских интерфейсов
  - Компонентный подход
  - Hooks для управления состоянием
  - Виртуальный DOM для оптимизации производительности

- **React Router DOM 7.1.3** - Библиотека для маршрутизации в React приложениях
  - Навигация между страницами
  - Защищенные маршруты
  - Программная навигация

- **Axios 1.7.9** - HTTP клиент для выполнения запросов к API
  - Перехватчики для автоматического добавления токенов
  - Обработка ошибок
  - Поддержка промисов

### Инструменты разработки

- **Vite 7.2.4** - Современный инструмент сборки
  - Быстрая разработка с HMR (Hot Module Replacement)
  - Оптимизированная сборка для продакшена
  - Поддержка ES модулей

- **ESLint 9.39.1** - Линтер для JavaScript/React
  - Проверка кода на ошибки
  - Следование стандартам кодирования
  - Правила для React hooks

## Backend

### Основные технологии

- **Node.js** - JavaScript runtime для серверной разработки
  - Асинхронный I/O
  - Масштабируемость
  - Большая экосистема пакетов

- **Express 5.2.1** - Минималистичный веб-фреймворк для Node.js
  - Маршрутизация
  - Middleware поддержка
  - RESTful API

- **PostgreSQL** - Реляционная база данных
  - ACID совместимость
  - Надежность
  - Расширяемость

- **Sequelize 6.37.3** - ORM для Node.js
  - Работа с базой данных через модели
  - Миграции
  - Ассоциации между моделями

### Безопасность

- **jsonwebtoken 9.0.2** - JWT токены для аутентификации
  - Stateless аутентификация
  - Безопасная передача данных пользователя
  - Истечение срока действия токенов

- **bcryptjs 2.4.3** - Хеширование паролей
  - Одностороннее хеширование
  - Соль для безопасности
  - Медленное хеширование для защиты от брутфорса

- **dotenv 17.2.3** - Управление переменными окружения
  - Безопасное хранение секретов
  - Конфигурация для разных окружений

### Работа с файлами

- **multer 1.4.5-lts.1** - Middleware для загрузки файлов
  - Загрузка файлов через multipart/form-data
  - Контроль размера файлов
  - Генерация уникальных имен

### Утилиты

- **cors 2.8.5** - Cross-Origin Resource Sharing
  - Разрешение запросов с других доменов
  - Настройка CORS политики

- **pg 8.16.3** - PostgreSQL клиент для Node.js
  - Нативный драйвер для PostgreSQL
  - Поддержка пула соединений

- **pg-hstore 2.3.4** - Поддержка hstore типа данных PostgreSQL

### Инструменты разработки

- **nodemon 3.1.11** - Автоматический перезапуск сервера при изменениях
  - Удобство разработки
  - Автоматическое обнаружение изменений

## База данных

### PostgreSQL

- **Версия**: Рекомендуется 12+
- **Особенности использования**:
  - Использование транзакций
  - Индексы для оптимизации запросов
  - Foreign keys для целостности данных
  - JSON поддержка для хранения структурированных данных

### Структура базы данных

Основные таблицы:
- `Users` - пользователи системы
- `Courses` - курсы
- `Assignments` - задания
- `AssignmentSubmissions` - решения заданий
- `Schedule` - расписание
- `Materials` - учебные материалы
- `Grades` - оценки
- `StudentCourses` - связь студентов и курсов

## Архитектура проекта

### Backend архитектура

```
backend/
├── config/          # Конфигурация (база данных, настройки)
├── controllers/     # Контроллеры (бизнес-логика)
├── middleware/      # Middleware (аутентификация, валидация)
├── migrations/      # Миграции базы данных
├── models/          # Модели Sequelize
├── routes/          # Маршруты API
└── server.js        # Точка входа приложения
```

### Frontend архитектура

```
frontend/
├── src/
│   ├── components/  # Переиспользуемые компоненты
│   ├── pages/       # Страницы приложения
│   ├── services/    # API сервисы
│   ├── context/     # React Context (состояние приложения)
│   └── App.jsx      # Главный компонент
└── public/          # Статические файлы
```

### Переменные окружения

Необходимые переменные для backend (.env):
- `DB_USER` - пользователь базы данных
- `DB_HOST` - хост базы данных
- `DB_NAME` - имя базы данных
- `DB_PASSWORD` - пароль базы данных
- `DB_PORT` - порт базы данных
- `JWT_SECRET` - секретный ключ для JWT
- `PORT` - порт сервера
- `NODE_ENV` - окружение (development/production)

## Производительность и оптимизация

### Backend

- Кэширование запросов (если необходимо)
- Индексы в базе данных
- Оптимизация SQL запросов через Sequelize
- Сжатие ответов (gzip)
- Ограничение размера загружаемых файлов

### Frontend

- Code splitting для уменьшения размера бандла
- Lazy loading компонентов
- Оптимизация изображений
- Минификация CSS и JavaScript в продакшене

## Безопасность

### Меры безопасности

1. **Аутентификация**: JWT токены
2. **Хеширование паролей**: bcrypt
3. **Валидация данных**: на клиенте и сервере
4. **CORS**: настройка разрешенных источников
5. **SQL инъекции**: защита через ORM
6. **XSS**: санитизация пользовательского ввода
7. **CSRF**: защита через токены (можно добавить)
