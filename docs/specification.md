# Техническое задание: Веб-приложение "Студенческий портал"

## 1. Описание предметной области

Студенческий портал — это веб-приложение, предназначенное для организации учебного процесса и управления студенческой жизнью. Система предоставляет единую платформу для студентов и преподавателей, позволяющую эффективно управлять учебными материалами, заданиями, расписанием и оценками.

### 1.1. Целевая аудитория

- **Студенты**: Основные пользователи системы, имеющие доступ к учебным материалам, заданиям, расписанию и своим оценкам
- **Преподаватели**: Создают курсы, выдают задания, загружают материалы, проверяют работы и выставляют оценки
- **Администраторы**: Управляют системой, пользователями и курсами

## 2. Функциональные требования

### 2.1. Система аутентификации и авторизации

#### 2.1.1. Регистрация
- Регистрация новых пользователей (студентов и преподавателей)
- Валидация email и пароля
- Хеширование паролей
- Разделение по ролям (student, teacher, admin)

#### 2.1.2. Вход в систему
- Аутентификация по email и паролю
- Генерация JWT токенов
- Сохранение сессии пользователя

#### 2.1.3. Авторизация
- Защита маршрутов с помощью middleware
- Проверка прав доступа на основе ролей
- Автоматический выход при истечении токена

### 2.2. Личный кабинет пользователя

#### 2.2.1. Профиль студента/преподавателя
- Просмотр основной информации
- Редактирование ФИО и группы (для студентов)
- Просмотр роли и email

#### 2.2.2. Академическая успеваемость (для студентов)
- Просмотр всех оценок
- Средний балл по курсам
- Успеваемость по каждому курсу отдельно

### 2.3. Управление курсами

#### 2.3.1. Создание курсов (преподаватели/админы)
- Название и описание курса
- Уникальный код курса
- Количество кредитов

#### 2.3.2. Запись на курсы (студенты)
- Просмотр доступных курсов
- Запись на курс
- Просмотр своих курсов

### 2.4. Система домашних заданий

#### 2.4.1. Выдача заданий (преподаватели)
- Создание задания с описанием
- Установка срока сдачи
- Максимальный балл
- Прикрепление файлов

#### 2.4.2. Сдача заданий (студенты)
- Отправка решений заданий
- Прикрепление файлов
- Просмотр статуса проверки

#### 2.4.3. Проверка заданий (преподаватели)
- Просмотр всех сданных решений
- Выставление оценки
- Добавление комментариев/обратной связи

### 2.5. Расписание занятий

#### 2.5.1. Просмотр расписания
- Расписание на неделю
- Фильтрация по дню недели
- Группировка по дням

#### 2.5.2. Управление расписанием (преподаватели/админы)
- Создание записей в расписании
- Указание курса, времени, аудитории, группы
- Редактирование и удаление записей

### 2.6. Учебные материалы

#### 2.6.1. Загрузка материалов (преподаватели)
- Загрузка файлов (документы, видео, ссылки)
- Описание материала
- Привязка к курсу

#### 2.6.2. Доступ к материалам
- Просмотр материалов по курсам
- Скачивание файлов
- Фильтрация по типу материала

### 2.7. Электронный журнал успеваемости

#### 2.7.1. Просмотр оценок (студенты)
- Все оценки по курсам
- Средний балл
- История оценок

#### 2.7.2. Управление оценками (преподаватели)
- Выставление оценок за задания
- Добавление обратной связи
- Просмотр успеваемости студентов

## 5. Схема базы данных

### 5.1. Основные сущности

- **Users**: Пользователи системы (студенты, преподаватели, админы)
- **Courses**: Курсы
- **StudentCourses**: Связь студентов с курсами (многие-ко-многим)
- **Assignments**: Задания
- **AssignmentSubmissions**: Решения заданий
- **Schedule**: Расписание занятий
- **Materials**: Учебные материалы
- **Grades**: Оценки

### 5.2. Связи между сущностями

- User (1) → (N) Assignment (teacher_id)
- User (1) → (N) AssignmentSubmission (student_id)
- User (1) → (N) Grade (student_id)
- User (1) → (N) Schedule (teacher_id)
- User (N) ↔ (N) Course (через StudentCourses)
- Course (1) → (N) Assignment
- Course (1) → (N) Material
- Course (1) → (N) Schedule
- Assignment (1) → (N) AssignmentSubmission
- AssignmentSubmission (1) → (1) Grade

## 6. API Specification

### 6.1. Аутентификация
- POST `/api/auth/register` - Регистрация
- POST `/api/auth/login` - Вход
- POST `/api/auth/logout` - Выход

### 6.2. Профиль
- GET `/api/profile/me` - Получить профиль
- PUT `/api/profile/me` - Обновить профиль

### 6.3. Курсы
- GET `/api/courses` - Все курсы
- GET `/api/courses/my` - Мои курсы
- GET `/api/courses/:id` - Курс по ID
- POST `/api/courses` - Создать курс (teacher/admin)
- POST `/api/courses/:courseId/enroll` - Записаться на курс (student)

### 6.4. Задания
- GET `/api/assignments` - Все задания
- GET `/api/assignments/my` - Мои задания
- GET `/api/assignments/:id` - Задание по ID
- POST `/api/assignments` - Создать задание (teacher)
- POST `/api/assignments/:id/submit` - Отправить решение (student)
- POST `/api/assignments/submissions/:submissionId/grade` - Оценить решение (teacher)

### 6.5. Расписание
- GET `/api/schedule` - Расписание
- GET `/api/schedule/:id` - Расписание по ID
- POST `/api/schedule` - Создать запись (teacher/admin)
- PUT `/api/schedule/:id` - Обновить запись
- DELETE `/api/schedule/:id` - Удалить запись

### 6.6. Материалы
- GET `/api/materials` - Все материалы
- GET `/api/materials/:id` - Материал по ID
- POST `/api/materials` - Создать материал (teacher)
- PUT `/api/materials/:id` - Обновить материал
- DELETE `/api/materials/:id` - Удалить материал

### 6.7. Оценки
- GET `/api/grades` - Все оценки
- GET `/api/grades/:id` - Оценка по ID
- GET `/api/grades/performance/:studentId?` - Успеваемость студента

7. Оценки отображаются и выставляются правильно
